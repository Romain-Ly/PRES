
La première partie est de consuitre les topologies de différents
réseaux virtuels et de tester les performances de MTPCP en faisant
varier les paramètres des sous-flots. La seconde partie est de
construire un algorithme d'ordonnancement répondant à un critère de
sécurité simple: rendre le \emph{sniffing} des paquets plus
difficile à réaliser.

\vspace{0.5cm}

Les étapes du développement suivront les points suivants:

\begin{itemize}
\item Préparation d'une machine mininet avec le noyau MPTCP compilé
  pour l'ensemble de l'équipe:
\item Lecture, compréhension et commentaires du code de MPTCP;
\item Préparation de plusieurs topologies : une topologie simple à
  plusieurs n\oe uds et un \emph{fat tree} pour simuler un \emph{data
    center};
\item Préparation d'une bibliothèque de tests et de mesures via l'API python;
\item \'Ecriture d'un algorithme d'ordonnancement dans le noyau;
\item Mesures de performance.
\end{itemize}

\vspace{1cm}

Le développement sera divisé en trois sous-projets:

\paragraph{Scripts de tests mininet} Le but de ce sous-projet est
d'établir une base pour tester les performances de MPTCP en fonction
des paramètres du réseau virtuel. Cette base sera réutilisée pour les
topologies plus complexe (sous-projet \emph{fat-tree}) ou pour la
mesure de performances du nouvel algorithme
d'ordonnancement. (sous-projet algorithme d'ordonnancement).
